Architecture Explained

The CalWise application follows a classic client-server architecture, composed of a React single-page application (SPA) for the frontend and a Node.js server for the backend API.

1. Frontend (Client)

The frontend is a modern React application built to provide a dynamic and responsive user experience.

  • Core Framework: React is used for building the user interface with a component-based structure.
  • Routing: React Router DOM (react-router-dom) handles all client-side routing. The ProtectedRoute component acts as a gatekeeper, using an AuthContext to verify if a user is authenticated before allowing access.
  • State Management: A React Context (AuthContext) is dedicated to managing the user's authentication state globally. It holds the user's data, authentication status, and provides login and logout functions that interact with the backend.
  • UI and Styling: The UI is built with Tailwind CSS for a utility-first styling approach, react-icons for icons, and react-toastify for notifications.

2. Backend (Server)

The backend is a Node.js application serving a REST API, likely built with a framework like Express.js.

  • Technology: Node.js is used to run the server, which listens on port 5000.
  • API Layer: It exposes a REST API to handle business logic, including endpoints for authentication (/api/auth/login) and data operations (/api/meals/upload).
  • Database: The backend connects to a database (like SQLite as mentioned in the README) to persist user data, meal history, and other application information.

3. Key Interaction Flows

Authentication Flow

  1. The user submits credentials on the Login page.
  2. The frontend sends a POST request to the backend's /api/login endpoint.
  3. The backend validates the credentials against the database.
  4. Upon success, it generates a JSON Web Token (JWT) and sends it back to the client.
  5. The frontend saves the token and updates its state, marking the user as authenticated and granting access to protected routes.

Data Fetching for Protected Routes

  1. When a user navigates to a protected page like /dashboard, the frontend first confirms they are authenticated.
  2. The page component then makes an API call (e.g., GET /api/meals) to the backend, sending the JWT for authorization.
  3. The backend verifies the JWT, fetches the required data from the database, and returns it to the frontend to be displayed.
